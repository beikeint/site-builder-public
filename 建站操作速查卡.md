# 建站操作速查卡（运营人员专用）

> 新客户来了，按这个顺序操作，每步一句指令。

---

## 第0步：录入客户

```
把客户资料填入 client-manager
```
→ 会生成 client-xxx ID，后续所有操作用这个 ID

## 第1步：启动建站

```
[客户名] 启动建站
```
→ **建站启动技能**自动执行：
  - 从 client-manager 读取客户信息
  - 发送素材清单（如果还没收到素材）
  - 抓取3-5个同行竞品网站
  - 基于竞品数据设计首页布局方案
  - 选择行业蓝图（heavy-equipment / consumer-goods / building-materials / electronics / food-medical）
  - 创建项目文件夹 + site.config.ts
  - **需要你确认布局方案后才继续**

## 第2步：抓取素材（如需要）

```
[客户名] 抓取素材
```
→ **内容抓取技能**：从客户旧站/阿里/1688/竞品抓取产品信息和图片

## 第3步：开始建站

```
[客户名] 开始建站
```
→ **建站执行技能**自动执行8阶段：
  1. 布局实施（按蓝图配置首页模块）
  2. 产品录入（products.ts）
  3. 图片处理（下载→压缩→WebP）
  4. 多语言翻译
  5. 博客（至少3篇）
  6. llms.txt + robots.txt
  7. SEO/转化检查
  8. 构建打包

## 第4步：质量检查

```
QA检查
```
→ `npm run qa` 15项自动检查 + seo-checker 审计

## 第5步：部署上线

```
部署 [客户名]
```
→ deployer 自动 build + 上传 + 验证

## 第6步：交付验收

```
[客户名] 交付验收
```
→ 竞品对标打分 + 博客质量复核 + 上线检查清单

## 第7步：运营交接

```
[客户名] 交接运营
```
→ 基线报告 + 交接文档 + 写入 client timeline

---

## 快速参考

| 场景 | 指令 |
|------|------|
| 新客户从零开始 | 第0→7步全走 |
| 客户已有旧站改造 | 第0步 → 抓取素材 → 第1步起 |
| 客户只改内容/加产品 | `[客户名] 扩展网站` |
| 部署后优化 | `[客户名] 全面优化` 或 `[客户名] 行业强化` |
| 处理产品图片 | `处理图片 [目录]` |

## 客户需要提供的最低限度

**没有这些别开工：**
1. 公司英文名
2. 联系邮箱 + WhatsApp号码
3. 产品列表（名称+型号+规格，至少5个）
4. 域名

**有了更好：**
- 产品图片、工厂照片
- 认证证书
- 阿里巴巴/1688店铺链接（可以替代产品资料）
