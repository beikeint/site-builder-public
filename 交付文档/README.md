## 🚦 根据场景找文档

### 场景 A：**开始一个新客户项目**
1. 先发 [../新客户资料收集表.md](../新客户资料收集表.md) 或 [../新客户资料收集表-微信版.txt](../新客户资料收集表-微信版.txt) 给客户填
2. 收集齐资料后用 ../CLAUDE.md 定义的流程开工
3. 每个客户目录下放 `客户启动清单.md` / `竞品布局分析.md` / `素材整理.md`（参考 `客户版/客户E/`）

### 场景 B：**网站建完、要交给客户**
1. 把 [客户操作教程-通用模板.md](客户操作教程-通用模板.md) 复制一份到客户项目的 `docs/` 目录
2. **替换 `{{VARIABLE}}` 占位符**（域名、品牌、服务器、联系方式等）
3. 发给客户，让他按步骤注册 Web3Forms / GA4 / GSC / IndexNow 等
4. 客户发回 Key 后，运营人员填入 `site.config.ts` 并重新部署

### 场景 C：**客户站上线后日常运营**
看 [上线后运维SOP.md](上线后运维SOP.md)

### 场景 D：**想看某个客户现在状态/服务器/域名**
看 [已交付客户速查表.md](已交付客户速查表.md)

### 场景 E：**客户问技术问题**
客户操作相关答复翻 [客户操作教程-通用模板.md](客户操作教程-通用模板.md) 末尾的 FAQ 章节。

## ✅ 建站完整时序（从0到运营）

```
阶段1. 开工前       ──  发收集表 → 收资料 → 竞品分析
阶段2. 搭建中       ──  按CLAUDE.md流程，配置驱动 site.config.ts
阶段3. 测试部署     ──  12项QA必检（见CLAUDE.md）
阶段4. 交付客户     ──  发 客户操作教程（这里就是核心资产）
阶段5. 客户侧补充   ──  客户发回 Web3Forms/GA4/GSC/IndexNow Key
阶段6. 二次部署     ──  运营人员填Key+Key相关验证码 → 重新部署 → 客户验收
阶段7. 运营闭环     ──  每日巡检+每周增长+每月GSC+GA4报告
```
