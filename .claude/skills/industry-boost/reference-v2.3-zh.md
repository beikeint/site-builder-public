> 适用场景：针对中国制造商/工厂的B2B外贸独立站，应用行业最佳实践，让网站从"能用"变成"好用、能转化"。

> 更新时间：2026-04（v2.3）→ **v2.4 升级**

## 🚨 v2.4 升级补丁

### 从"默认全强化"到"叙事认可才强化"

**v2.3 错法**：接手就套"B2B 最佳实践清单"——首页结构强化 / 产品页强化 / FAQ 加强 / 信任体系建设 / GEO 优化 —— **不管客户业务模型是否匹配**，一股脑全上。结果给 Solution Integrator 强化"Pure Manufacturer 标准"→ 定位错位。

**v2.4 对法**：先读叙事档案，只对档案认可保留的 section 做深度强化；档案里砍掉的 section 不强化、不补；叙事单元承载的 section 重点强化。

### 必读前置

`客户/独立站-客户XXX/业务叙事档案.md` — 特别是第 3 节（蓝图 section 保留/删除决策）和第 4 节（最终页面地图）。

### 强化原则（v2.4）

| 场景 | v2.3 行为 | v2.4 行为 |
|---|---|---|
| 档案保留的 section | 强化 | 强化（优先深度强化叙事单元承载的）|
| 档案删除的 section | 强化 | **不碰**（档案删了就是删了，不能私自补回来）|
| 档案没说的 section | 默认强化 | **先回 build-startup 确认**，不私自添加 |

### 行业 × 业务模型强化矩阵（新增）

原来只有"行业"一维。v2.4 加上"业务模型"维度：

| 行业 | 业务模型 | 强化重点（不同）|
|---|---|---|
| 冷链设备 | Pure Manufacturer | 产品参数表 / 产能 / 认证 |
| 冷链设备 | **Solution Integrator** | **按买家类型方案 / 供应网络 / 整合能力** ← Demo-A 正确方向 |
| 化工 | Pure Manufacturer | 产品规格 / MSDS / TDS / 批量 |
| 化工 | **Material Supplier** | **应用场景案例 / 下游成品图 / 样品流程** ← Demo-B 方向 |

即使同一行业，业务模型不同 → 强化重点完全不同。档案里的业务模型判定优先级高于行业蓝图。

### 硬规则（v2.4）

- ❌ **禁止未读叙事档案就开始"B2B 最佳实践全套上"**
- ❌ **禁止给档案删除的 section 补内容**（档案删了有理由，私自补 = 违抗叙事方向）
- ❌ **禁止把 Solution Integrator 客户按 Pure Manufacturer 强化**（Demo-A 事件正是这样错位的）

### GEO 强化（不变，仍然必做）

DNA 层的 GEO 答案胶囊、FAQPage Schema、llms.txt 等**不受 v2.4 影响**，继续满配。

## 零、行业类型判断（开始前必读）

**使用本技能前，先判断客户属于哪种行业类型，两种类型的建站策略有本质区别：**

### 类型A：B2B制造商（产品驱动型）

**典型行业：** PPE/安全防护、电子元件、建材五金、化工材料、食品设备、服装纺织、日用消费品
**客户特征：** 产品SKU多（10-100+个）、按产品分类浏览、买家按型号/规格采购
**网站策略：**
- 5语言支持（英/西/法/阿/俄）+ 阿拉伯语RTL
- 以产品目录为核心，分类清晰，详情页规格参数完整
- 页面结构：首页 / 产品列表 / 产品详情 / 关于 / 质量 / 博客 / 联系
- 标准模板直接适用

### 类型B：B2B工业设备商（方案驱动型）

**典型行业：** 工业机械、自动化设备、生产线、环保设备、能源设备、检测仪器
**客户特征：** 产品SKU少但单价高（几万到几百万）、买家按"解决什么问题"或"用在什么行业"来找产品
**网站策略：**
- 3语言支持（英/西/法）— 工业设备买家以欧美为主，阿语俄语优先级低
- 以解决方案为核心，产品页面是方案的支撑
- 额外必须页面：
  - **Solutions页面** — 按场景组织（如"噪音控制方案"、"粉尘处理方案"、"自动化升级方案"），每个方案关联对应产品组合 + 应用案例
  - **Applications页面** — 按行业组织（如"Mining"、"Oil & Gas"、"Food Processing"），展示该行业适用的产品 + 成功案例
- 页面结构：首页 / Solutions / Applications / Products / 关于 / 质量 / 博客 / 联系
- 需要在标准模板基础上扩展 Solutions 和 Applications 页面

### 如何判断？

| 判断依据 | 类型A（制造商） | 类型B（设备商） |
|---------|---------------|---------------|
| 产品单价 | < ¥1000/件 | > ¥10000/套 |
| SKU数量 | 10-100+ | 3-20 |
| 买家采购方式 | 按产品型号找 | 按问题/行业找 |
| 询盘内容 | "SA-7-1报价多少" | "我们工厂噪音超标，有什么方案" |
| 决策周期 | 1-4周 | 1-6个月 |
| 语言优先级 | 5语言（全球覆盖） | 3语言（欧美重点） |

**判断完成后，在后续所有提示词中注明类型（A或B），Claude 会按对应策略执行。**

## 一、使用场景

- 建站基本完成，需要按B2B制造商行业标准进行强化
- 客户是工厂/制造商，目标客户是海外采购商/经销商
- 需要在SEO、转化、信任建设等方面达到行业领先水平
- 针对特定行业（PPE/机械/电子/建材等）做定制化内容

---

## 二、B2B制造商网站核心原则

1. **信任第一** — 海外采购商最关心的是"你是不是真实可靠的工厂"
2. **询盘为王** — 所有设计都服务于一个目标：让客户提交询盘
3. **内容即销售** — 产品规格、认证、案例就是最好的销售话术
4. **多语言覆盖** — 不只是英文，要覆盖目标市场的本地语言
5. **AI可发现** — 让ChatGPT、Perplexity等AI搜索工具能推荐你

## 五、注意事项

1. **先判断行业类型再动手** — 类型A和类型B的策略差异很大，搞错了等于返工
2. **信任建设需要真实素材** — 认证证书、工厂照片、客户案例不能造假，缺失就标注"待客户提供"
3. **数据展示要具体** — "10+ years" 比 "many years" 好，"200M+ units/year" 比 "large capacity" 好
4. **FAQ要像真人会问的** — 不要学术化，要像采购经理在微信里问的那种问题
5. **博客不是凑数** — 每篇文章都要解决采购商的一个实际问题
6. **GEO是新战场** — 2026年越来越多采购商用AI搜索找供应商，提前布局
7. **行业标准要准确** — CE EN352-2、ANSI S3.19 这些标准编号不能写错
8. **类型B的语言不要过多** — 工业设备3语言足够，多了反而增加维护成本且目标市场不匹配
9. **Solutions/Applications页面也要做SEO** — 每个方案页和行业页都需要独立title、description和Schema

## 六、v2.3 新增强化项

> 以下内容基于 astro-b2b-starter v2.3 架构，在行业强化阶段一并完成。

### 6.1 产品 wizardTags 填写（选型向导数据）

如果 `features.productWizard = true`，必须为每个产品填写 `wizardTags`：

**类型A（制造商）常用维度：**
```typescript
wizardTags: {
  material: "silicone",        // 材质
  application: "kitchen",      // 应用场景
  certification: "FDA",        // 认证
  moq: "500+",                 // 最低起订量区间
}
```

**类型B（设备商）常用维度：**
```typescript
wizardTags: {
  capacity: "5-20 tons/day",   // 产能范围
  power: "7.5-15 kW",         // 功率范围
  automation: "semi",          // manual / semi / full
  budget: "mid",               // low / mid / high
}
```

维度选择原则：选择采购商最关心的 3-4 个决策维度，每个维度的值不超过 5 个选项。

### 6.2 ROI 计算器行业参数（roi-config.ts）

如果 `features.roiCalculator = true`，必须填写 `src/data/roi-config.ts`：

```typescript
formula: {
  investmentPerUnit: 1500,       // 每单位产能的设备投资（$/unit）— 按行业实际填
  laborSavingRate: 0.40,         // 自动化后人工节省比例 — 按行业实际填
  productValuePerUnit: 200,      // 每单位产品价值（$/unit）— 按行业实际填
  maintenanceCostRate: 0.03,     // 运维年成本占比 — 按行业实际填
}
```

如果客户无法提供准确参数，使用行业平均值并在计算结果页注明"基于行业平均估算"。

### 6.3 首页 FAQ 内容定制

starter 自动生成 6 个首页 FAQ（从 config 和产品数据），但内容是通用模板。行业强化阶段需要：

1. 检查自动生成的 FAQ 答案是否准确
2. 替换通用表述为行业特定内容
3. 确保 FAQ 答案包含具体数据（产能、年限、认证编号）
4. 确保 FAQ 答案能被 AI 搜索提取（直接回答、结构化数据）

### 6.4 产品页 FAQ 检查

每个产品详情页自动生成 4 个 FAQ。行业强化阶段需要：

1. 检查"What is X used for"的回答是否准确反映该行业的实际应用
2. 检查规格参数回答是否完整
3. 检查认证回答是否列出了行业相关的所有认证
4. 确保 FAQ 有对应的 FAQPage Schema（自动注入，但验证一下）

### 6.5 GEO 答案胶囊内容优化

产品页的答案胶囊自动生成，但行业强化阶段需要确保：

1. 前 60 词包含：产品名 + 型号 + 认证 + 核心参数 + 应用场景
2. 表述方式是"定义性语句"（"X is a..."），而不是营销文案
3. 参数使用具体数值，不用模糊表述
4. 检查 `<table>` 规格表是否完整（AI 搜索最容易提取表格数据）

### 6.6 GA4 事件验证

确认以下事件在行业强化后仍然正常触发：

```
□ 产品页 product_view 自动触发
□ 产品页内嵌询盘 form_start / form_submit
□ 浮动 CTA cta_click
□ WhatsApp whatsapp_click
□ 首页 FAQ 展开（通过滚动深度间接追踪）
```

