# 内容抓取技能 — 从客户现有渠道提取建站素材

> 适用场景：客户提供了官网、阿里巴巴国际站、1688店铺等链接，需要从中提取产品信息、公司资料、图片等建站素材。
> 使用者：独立站建站智能体（内部）
> 更新时间：2026-03-31

---

## 一、使用场景

- 客户有现成的官网/阿里巴巴/1688店铺，但素材散落各处
- 客户无法提供完整的产品规格表，需要从现有渠道提取
- 需要快速整理出建站所需的全部素材清单
- 客户提供了竞品网站作为参考

---

## 二、抓取流程

### 第1步：确认数据源

向客户确认可用的数据源：

| 数据源 | 可获取内容 | 优先级 |
|--------|-----------|--------|
| 客户官网 | 公司简介、产品列表、联系方式、证书 | 高 |
| 阿里巴巴国际站 | 产品详情、规格参数、图片、公司资质、年限 | 高 |
| 1688 店铺 | 中文产品信息、价格区间、规格参数 | 中 |
| Made-in-China | 英文产品信息、公司认证 | 中 |
| 参考/竞品网站 | 页面结构、内容风格、SEO策略 | 参考 |

### 第2步：逐源抓取

按优先级依次抓取，每个源使用对应提示词。

### 第3步：整合为素材整理文档

将所有抓取内容整合为统一格式的 `素材整理.md`，标注数据来源和待补充内容。

---

## 三、核心提示词

### 提示词1：从客户官网抓取素材

```
请访问以下客户官网，提取建站所需的全部素材：

官网地址：【URL】

请按以下结构提取并整理：

一、公司信息
- 中文名、英文名、成立时间、地址（中英文）
- 公司简介（原文摘录）
- 产能数据（年产量、出口比例等）
- 认证资质（CE、ISO、FDA、BSCI等）
- 公司里程碑/发展历程

二、联系方式
- 联系人姓名（英文）、职位
- 电话、邮箱、WhatsApp
- 公司地址

三、产品信息（每个产品一条）
- 产品名称（中英文）、型号
- 所属分类
- 材质、规格参数（尺寸、重量、标准等）
- 产品特点/卖点
- 应用场景
- 认证标准
- 图片URL

四、服务能力
- OEM/ODM、定制包装、快速交付等

五、现有图片资源
- 列出可用的产品图片URL和格式

六、待补充内容清单
- 列出官网缺失但建站需要的信息（工厂照片、团队照片、证书扫描件等）

输出格式：Markdown表格，便于后续使用。
```

- 公司认证状态
- 主营产品类目

- 工厂面积、产线数量

二、产品列表（按分类整理）
每个产品提取：
| 字段 | 内容 |
|------|------|
| 产品名称 | |
| 型号 | |
| 分类 | |
| 材质 | |
| 规格参数 | （用 key: value 格式） |
| 产品特点 | |
| 认证 | |
| MOQ | |
| 图片URL | |

三、公司介绍页内容
- Company Profile 原文

### 提示词3：从1688店铺抓取补充信息

```
请访问以下1688店铺，提取官网和阿里站未覆盖的信息：

1688店铺地址：【URL】

重点提取：
1. 产品中文名称（用于确认翻译准确性）
2. 详细规格参数（1688通常比国际站更详细）
3. 产品分类方式（可能与国际站不同）
4. 价格区间（内部参考，不上网站）
5. 客户评价/成交记录（了解热销产品）

输出：仅列出与之前抓取不同或补充的信息，不重复已有内容。
```

### 提示词4：从竞品网站提取参考信息

```
请分析以下竞品/参考网站，提取值得借鉴的内容和结构：

参考网站：【URL】

请分析：

一、网站结构
- 导航结构（列出所有页面）
- 产品分类方式
- URL结构

二、内容策略
- 首页包含哪些模块（按顺序列出）
- 产品详情页包含哪些内容
- 是否有博客/文章
- FAQ内容方向

三、转化设计
- 询盘入口有几个，分别在哪里
- CTA按钮的文案和位置
- 是否有浮动按钮/在线聊天
- 信任背书元素（认证、客户数、产能等）

四、SEO策略
- Title/Description 写法
- 是否有Schema结构化数据
- 多语言支持情况
- 关键词布局

五、可借鉴的点（Top 5）
- 列出最值得参考的5个设计/内容/功能点

六、我们可以做得更好的地方
- 列出竞品的不足，我们可以超越的点
```

### 提示词5：整合素材为统一文档

```
请将以下多个来源的素材整合为一份统一的「素材整理.md」文档：

来源1（官网）：
【粘贴官网抓取结果】

来源2（阿里巴巴）：
【粘贴阿里抓取结果】

来源3（1688，如有）：
【粘贴1688抓取结果】

整合要求：
1. 按统一格式输出：公司信息 → 产品信息 → 服务能力 → 品牌定位 → 图片资源 → 待补充清单
2. 信息冲突时，以阿里国际站为准（面向海外市场）
3. 产品信息用表格格式，按分类分组
4. 在文档顶部注明数据来源和抓取日期
5. 底部列出「待补充内容清单」，分为「必须补充」和「建议优化」两类
6. 产品图片URL统一列出，标注分辨率是否满足建站需求（≥800px宽）
```

## 五、注意事项

1. **先抓取后联系客户** — 能从现有渠道获取的信息先自己抓，减少客户填写负担
2. **图片质量评估** — 抓取时标注图片大小，<30KB 的图片在详情页会模糊，需客户补拍
3. **规格参数是核心** — 产品规格参数表格是详情页最重要的内容，务必尽量完整
4. **标注数据来源** — 每条信息标注来自哪个渠道，方便后续核实
5. **待补充清单要具体** — 不要写"需要更多信息"，要写"SA-7-1 缺少 SNR/NRR 数据"
6. **不要凭空编造数据** — 抓取不到的参数留空，标注"待客户确认"，绝不虚构
7. **尊重版权** — 竞品网站的内容只做参考分析，不复制文案
