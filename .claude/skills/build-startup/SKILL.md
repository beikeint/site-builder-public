# 建站启动技能 — 从零开始到项目就绪

> 适用场景：接到新客户建站需求时，从需求确认到项目文件夹搭建、CLAUDE.md 配置的完整启动流程。
> 使用者：独立站建站智能体（内部）
> 更新时间：2026-03-31

---

## 一、使用场景

- 新客户签约后，启动独立站建设项目
- 客户提供了基本信息（公司名、行业、参考网站等）
- 需要快速搭建项目骨架，配置好 Claude 工作环境

---

## 二、启动流程（按顺序执行）

### 第1步：需求确认

向客户确认以下信息（微信/邮件均可）：

| 必填项 | 说明 |
|--------|------|
| 公司中文名 + 英文名/品牌名 | 用于网站全局 |
| 主营产品（大类） | 决定网站结构和配色 |
| 参考网站（如有） | 用于抓取素材和风格参考 |
| 联系人 + 邮箱 + 电话 + WhatsApp | 网站联系页 + 询盘接收 |
| 域名（如已注册） | 用于 SEO 配置 |
| 目标市场 | 决定语言和内容方向 |

**如果客户信息不全，发送素材收集清单（见下方提示词）。**

### 第2步：创建项目文件夹 + 注册客户

**确定客户编号**：读取 `${WORKSPACE_ROOT}/客户/` 目录，找到最大编号+1（如已有 001、002、003 则新客户为 004）。

**创建项目目录**：

```
在 ${WORKSPACE_ROOT}/客户/ 下创建：
独立站-客户XXX/
├── CLAUDE.md          ← 项目配置（Claude 工作指令）
├── 素材整理.md         ← 抓取/整理的客户素材
├── website/           ← 网站源码（从模板复制）
```

**注册到 client-manager**：

```
client-manager.add_client({
  company_cn: "公司中文名",
  company_en: "英文名/品牌名",
  industry: "行业",
  products: "主营产品",
  domain: "域名",
  target_market: "目标市场",
  plan: "服务方案",
  status: "active",
  notes: "本地目录：客户/独立站-客户XXX"
})
```

> 建站技能已全局挂载在 `智能体/建站/独立站建站-site-builder/.claude/skills/`，无需复制到客户项目目录。

### 第3步：配置 CLAUDE.md

### 第4步：发送素材收集清单（如客户未提供完整素材）

### 第5步：确认启动

与客户书面确认以下事项后，进入建站执行阶段：
- 网站定位确认（B2B询盘型）
- **语言版本确认（根据目标市场判断，不固定5种语言）**
- 产品分类确认
- 参考网站/风格偏好确认

> **语言方案说明：**
> 不要默认所有客户都做5种语言（英/西/法/阿/俄），要根据目标市场实际判断：
> - 目标市场是欧美 → 英语为主，可加西/法/德
> - 目标市场是中东 → 英语 + 阿拉伯语
> - 目标市场是东南亚 → 英语即可，部分市场可加泰语/越南语
> - 目标市场是全球 → 英语 + 3-5种覆盖主要市场的语言
> - 语言数量影响工作量和维护成本，要跟客户沟通确认，不要自作主张

## 三、核心提示词

### 提示词1：创建项目 CLAUDE.md

```
请为新客户创建独立站项目的 CLAUDE.md 配置文件，包含以下信息：

客户信息：
- 公司：【中文名】
- 英文名/品牌：【英文名】
- 地址：【地址】
- 主营产品：【产品类型】
- 联系人：【姓名】
- 邮箱：【邮箱】
- 电话：【电话】
- WhatsApp：【号码】

网站定位：
- 类型：B2B询盘型独立站
- 目标市场：【如：全球B端、欧美重点】
- 参考网站：【URL】
- 域名：【域名】

请按以下结构输出 CLAUDE.md：

## 客户信息
（公司名、地址、主营产品、其他业务、资质）

## 目标市场
（主要市场、重点市场）

## 网站定位
（主要目的、展示内容、面向对象）

## 多语言要求
（5语言表格：英/西/法/阿/俄，标注RTL）

## 参考网站
（URL + 参考说明）

## 开发注意事项
（客户无技术团队、方案简单、RTL支持、询盘突出、产品分类清晰）

## SEO基础要求
（title/description、Schema.org、图片alt、WebP、sitemap、robots.txt、URL结构、title格式）

## 项目进展
（记录关键节点和客户确认事项）
```

### 提示词2：发送素材收集清单给客户

```
请生成一份给客户填写的素材收集清单，中文，简洁易懂，包含：

一、公司基本信息（必填）
- 公司名称（中英文）、成立年份、公司简介、主营产品、年产能、工厂地址、出口情况

二、联系信息（必填）
- 业务邮箱、电话、WhatsApp、地址、联系人

三、产品信息（必填）
- 每个产品：名称、型号、分类、图片(2-3张白底高清)、规格参数、描述/卖点、认证、适用场景、是否热销/新品、MOQ、OEM/ODM
- 产品分类列表

四、证书资质（强烈建议）
- ISO 9001、CE、FDA、SGS、BSCI 等扫描件

五、视觉素材（有则提供）
- Logo、工厂照片、车间照片、团队照片、产品场景照

六、品牌偏好（可选）
- 喜欢的同行网站、偏好色调、域名、特殊需求

七、SEO相关（可选）
- 客户搜索词、主要竞品网站、目标客户类型、阿里/MIC店铺链接

格式要求：用表格，留空让客户填写，底部注明提交方式和3个工作日启动承诺。
```

## 四、注意事项

1. **CLAUDE.md 是核心** — 它决定了 Claude 在后续建站中的所有行为，配置务必完整准确
2. **客户确认要书面** — 微信截图或邮件回复"确认"即可，避免口头确认后扯皮
3. **不要等素材齐全才启动** — 先用参考网站/阿里巴巴抓取素材，客户素材后期替换
4. **域名未确定时用占位符** — `example.com`，上线前统一替换
5. **每个项目独立文件夹** — 在 `${WORKSPACE_ROOT}/客户/` 下创建，命名格式：`独立站-客户XXX/`
6. **项目进展实时记录** — 在 CLAUDE.md 底部 `## 项目进展` 中记录每个关键节点
